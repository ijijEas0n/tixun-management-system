# 语音识别功能架构

## 已实现的固定架构

### 1. 录音控制

成绩录入页保留原来的开始、结束、取消和最长识别时间控制。开始录音时会清空上一轮待确认结果，避免旧结果误填入新一轮成绩。

### 2. 本地语音结束判断

系统会在腾讯云识别之外，额外启动本地音量检测：

- 长时间没有检测到有效说话，会自动结束；
- 检测到用户说过话后，如果持续静音，会自动结束；
- 自动结束失败时，用户仍然可以手动点“结束”。

这一层不替代语音识别，只负责判断“用户是不是说完了”。

### 3. 识别文本清洗

识别结果进入成绩解析前，会先做统一清洗：

- 把“一到、二到、三档、四导”等常见误识别纠正为“一道、二道、三道、四道”；
- 统一逗号、句号、分号；
- 统一“第几投”为“第几次”；
- 去掉影响解析的空格和重复标点。

### 4. 当前分组词典

每次语音录入时，系统会根据当前分组生成上下文词典，包含：

- 当前组学生姓名；
- 学号；
- 道次或顺序；
- 第一次、第二次、第三次；
- 项目常见词，比如 100 米、铅球、三级跳、800 米、犯规、备注等。

当前版本先在本地清洗和候选判断中使用这份词典。腾讯云如果要真正吃到动态热词，还需要后续接服务端热词表管理。

### 5. 候选结果确认

语音结果不会再全部直接写入成绩：

- 通过姓名或学号匹配到的结果，系统认为更可靠，可以自动填入；
- 只通过道次或顺序匹配到的结果，会进入“待确认”；
- 无法匹配的句子会显示在“未识别”里，不会写入成绩。

这样可以避免“一道、二道”被错听后直接污染成绩。

### 6. 可替换识别引擎边界

现在仍然使用腾讯云实时识别，但新增的清洗、确认、本地结束判断都不依赖腾讯云。后续可以把腾讯云替换为 Whisper、Groq、Deepgram 或其他识别服务，而不用重写成绩解析和确认流程。

## 需要人工补充的训练/评测素材

这些内容不能靠代码凭空生成，必须来自真实使用现场：

1. **真实操场录音**
   - 每个项目至少 20-50 条；
   - 包含男声、女声、远近不同、背景噪声、多人说话；
   - 覆盖短句、连续报成绩、带备注、说错后纠正。

2. **人工正确文本**
   - 每条录音都需要人工写出正确文本；
   - 例如：`一道十二秒八，二道十二秒六，李承泽备注起跑慢`。

3. **常见误识别词**
   - 例如：`一到 -> 一道`、`三档 -> 三道`、`十二秒吧 -> 十二秒八`；
   - 后续会加入清洗规则或热词表。

4. **学生姓名读法**
   - 容易错的姓名需要单独记录；
   - 尤其是多音字、同音名、两个字和三个字姓名。

5. **验收样本集**
   - 每次调整语音功能后，用同一批样本复测；
   - 记录识别准确率、断句准确率、成绩填入准确率。

## 下一步建议

第一轮现场测试优先收集“100 米短句连续报成绩”和“铅球/三级跳三次试投”两类样本。它们最容易暴露数字、道次、试次和自动结束的问题。
